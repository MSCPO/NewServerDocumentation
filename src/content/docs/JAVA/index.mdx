## 如何选择？

- **新手推荐**：Paper（性能好，文档多）
- **模组玩家**：Forge/NeoForge或Fabric/Quilt
- **小型联机**：Vanilla
- **大型服务器**：Paper或Pufferfish

## 服务器端简介

### Vanilla（原版）
官方原生服务端，兼容性最好，但扩展性有限。适合小型朋友联机服务器。

### Spigot系
基于CraftBukkit的服务端，支持插件，是Java版最流行的服务端系列。包括：
- Spigot：高性能服务端
- Paper：Spigot的优化版本，性能更好
- Pufferfish：进一步优化的Paper分支
- Purpur：功能最丰富的Paper分支

### Forge/NeoForge
支持模组的服务端，可以运行大量模组。适合想要模组化游戏体验的玩家。

### Fabric/Quilt
轻量级模组加载器，启动速度快，模组生态活跃。

### Sponge
SpongeAPI服务端，提供独特的插件系统和API。

### Cuberite
用C++编写的服务端，性能优异，但生态相对较小。
